## 3. Ограничения и особенности языка

### 3.1 Python как интерпретируемый язык и что это значит на практике
Python выполняет код динамически: интерпретатор читает программу построчно, преобразует её в байткод и исполняет внутри виртуальной машины.  
Практические последствия:

- ошибки многих типов проявляются только во время выполнения;
- запуск кода происходит без компиляции, что ускоряет разработку;
- сам процесс выполнения медленнее по сравнению с компилируемыми языками (C/C++, Rust), так как интерпретатор обрабатывает каждую операцию.

Интерпретируемость — это обмен: более высокая гибкость разработки в обмен на часть производительности.

### 3.2 GIL: глобальная блокировка интерпретатора
CPython использует GIL (Global Interpreter Lock) — механизм, который позволяет исполнять байткод только одному потоку за раз.  
Практические эффекты:

- многопоточные задачи, нагружающие **CPU**, не ускоряются пропорционально количеству потоков;
- задачи, связанные с ожиданием (ввод-вывод, сетевые операции), работают отлично — потоки по очереди получают управление, не блокируя всю программу;
- настоящий параллелизм для вычислений достигается через `multiprocessing`, `numba`, `cython`, вызовы нативного кода или использование библиотек, которые выполняют тяжёлые части вне GIL.

GIL — это не «ошибка языка», а архитектурный компромисс ради простоты и безопасности работы со ссылками и памятью.

### 3.3 Скорость, типизация и границы применимости
Python — динамически типизируемый язык. Типы проверяются во время выполнения, что:

- облегчает написание кода и прототипирование;
- усложняет оптимизацию производительности;
- делает ошибки возможными в рантайме, если тип оказался не тем, что ожидалось.

По скорости Python уступает системным языкам. Сильные стороны языка лежат не в «сырой производительности», а в:

- богатой экосистеме библиотек (NumPy, Pandas, PyTorch);
- удобной связке Python-код + высокопроизводительные C/C++-расширения;
- скорости разработки и читаемости.

Границы применимости хорошо видны:

- Python идеален для анализа данных, ML, скриптов, автоматизации, исследовательских прототипов, бэкенда.
- Для задач уровня «симуляция в миллиарды шагов на ядро» лучше использовать C/C++/Rust или специализированные решения.
- Для больших производственных систем важны типизация (mypy, Pydantic) и хорошая инженерная дисциплина.

Python — инструмент высокой выразительности, но не универсальный двигатель всего на свете. Его сила раскрывается там, где важнее гибкость и экосистема, чем предельная производительность.
