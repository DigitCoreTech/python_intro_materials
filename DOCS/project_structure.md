## 4. Модули: как устроены и откуда берутся

### 4.1 Что такое модуль и пакет
**Модуль** — это любой файл с кодом Python, обычно `module.py`.  
**Пакет** — это директория, содержащая набор модулей и файл `__init__.py`, который сообщает интерпретатору, что эту директорию можно импортировать.

Примеры:
- `import math` — импорт встроенного модуля.
- `import mypackage.utils` — импорт пользовательского пакета/модуля.

Модуль — это минимальная единица реиспользуемого кода. Пакет — способ группировать модули в логически связанную структуру.

### 4.2 Как Python ищет импортируемые вещи
Когда выполняется `import something`, Python проходит по списку путей `sys.path`:

1. Текущая директория.
2. Директории виртуального окружения.
3. Стандартная библиотека.
4. Установленные сторонние пакеты (`site-packages`).

Python ищет либо файл `something.py`, либо директорию `something` с `__init__.py`.  
Порядок важен: первый найденный вариант выигрывает. Поэтому конфликты имён — частая ошибка.

Посмотреть пути:
```python
import sys
print(sys.path)
```

### 4.3 Структура проекта на минималках

Типичная минимальная структура выглядит так:

```
project/
    mypackage/
        __init__.py
        core.py
        utils.py
    scripts/
        run.py
    requirements.txt
    README.md
```

Основные идеи:

* код лежит в пакете;
* скрипты используют пакет, а не копируют код;
* зависимости фиксируются в `requirements.txt` или `pyproject.toml`.
