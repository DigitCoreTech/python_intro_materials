# Введение: Как устроен Python и как его правильно установить

## 1. Как Python запускается и где вообще живёт

### 1.1 Что такое интерпретатор
Python — это интерпретируемый язык.  
Интерпретатор — это программа (`python` или `python3`), которая:

- читает ваш код строка за строкой,
- преобразует его во внутреннее представление (байткод),
- выполняет этот байткод внутри виртуальной машины Python (CPython VM).

Сам по себе `.py`-файл — просто текст. Всю «магию» делает интерпретатор.

### 1.2 Версии Python и зачем они важны
У Python параллельно существуют две ветки: условно **устаревшие** (2.x — уже не используются) и **актуальные** (3.x).

Версия важна, потому что:

- библиотеки могут поддерживать только определённые версии,
- синтаксис периодически меняется,
- разные проекты могут требовать разные версии.

Проверить установленную версию:
```bash
python3 --version
````

### 1.3 Как проходит путь от кода до выполнения

Упрощённая цепочка выглядит так:

1. Вы вызываете `python script.py`.
2. Интерпретатор считывает код → формирует **AST** (abstract syntax tree).
3. AST компилируется в **байткод** (`.pyc` файлы в папке `__pycache__`).
4. Байткод выполняется виртуальной машиной Python (CPython VM).
5. В процессе используются:

   * менеджер памяти Python,
   * сборщик мусора,
   * стек фреймов выполнения,
   * модули стандартной библиотеки.

## 2. Установка Python без боли

### 2.1 Где скачать и как проверить установку

Официальный источник:
[https://www.python.org/downloads/](https://www.python.org/downloads/)

После установки проверьте работоспособность:

```bash
python3 --version
pip3 --version
```

Если команда не находится, проблема, как правило, в PATH.

### 2.2 Отличия Windows / macOS / Linux

#### Windows

* Рекомендуется ставить через официальный установщик.
* Важно включить галочку **“Add python.exe to PATH”**.
* Команда запуска чаще всего `python`, а не `python3`.

#### macOS

* Лучше всего ставить через Homebrew:

  ```bash
  brew install python
  ```
* Система поставляет свой Python 2.x — использовать его не стоит.

#### Linux

* Python чаще всего уже установлен.
* Лучше ставить дополнительные версии через пакетный менеджер или `pyenv`.
* Важно понимать, что системный Python используется ОС — не ломаем его.

### 2.3 Как избежать типичных ловушек

Самые распространённые проблемы и их решения:

* **Несколько версий Python в системе**
  Используйте `which python` / `which python3`, чтобы понять, какой именно запускается.
  Рассмотрите установку с помощью `pyenv`, если требуется много версий.

* **PATH настроен некорректно**
  Python может быть установлен, но команду не видно.
  Добавьте путь к интерпретатору вручную или переустановите с правильными настройками.

* **pip устанавливает пакеты не туда**
  Часто возникает, когда `pip` и `python` принадлежат разным версиям.
  Проверяйте через:

  ```bash
  python3 -m pip install <package>
  ```

* **Смешивание системного Python и рабочего окружения**
  Всегда создавайте виртуальное окружение, чтобы зависимости не конфликтовали между проектами.
